## 开篇：这个策略要解决什么问题？

宠食记的 AI 给用户生成了一份鲜粮食谱——"鸡胸肉 80g、南瓜 60g、糙米 25g、蛋壳粉 1g"。然后呢？用户要自己打开盒马/永辉/叮咚买菜，一个一个搜、一个一个加购物车。食谱的精准到了克，但购买的体验还停在"自己搞定"。

我们要解决的，就是这一段。

**核心思路：宠食记不做供应链、不做仓储物流、不建自有电商。我们做的是"AI 帮你比价、帮你一站式跳转购买"。盒马、永辉超市、叮咚买菜、美团闪购——这些已经在你家附近有仓、有冷链、能 30 分钟送到家门口的平台——就是我们的"货架"。**

## 1. 为什么选择盒马和永辉这类商超，而不是淘宝京东？

鲜粮食谱的食材构成很奇怪——80g 鸡胸肉、60g 南瓜、25g 糙米。这不是标准电商包裹的规格。淘宝上你能买到一整箱鸡胸肉，但你只需要 80g。京东上你能买到 5kg 装的糙米，但你只需要 25g。

**鲜粮食谱的食材天然适合商超和即时零售，不适合传统电商。**

| 维度 | 淘宝/京东（传统电商） | 盒马/永辉/叮咚（即时零售） |
|------|------|------|
| 商品类型 | 标品（狗粮猫粮、宠物零食） | 生鲜（鸡胸肉、南瓜、胡萝卜、蓝莓）——鲜粮食谱的核心食材 |
| 配送时效 | 2-3 天 | 30 分钟~1 小时 |
| 购买单位 | 整箱/大包装 | 散装称重，可以只买 80g |
| 用户使用场景 | 囤货、补货 | "今天要做宠物饭，现在买食材" |
| 和鲜粮食谱的匹配度 | ⭐⭐ | ⭐⭐⭐⭐⭐ |

**结论：干粮/标品走淘宝京东联盟链接没问题。但鲜粮/自制食材——这是宠食记和所有其他宠物产品最大的差异点——必须走盒马/永辉这类商超渠道。**

## 2. 核心产品设计：AI 比价 → 一站式购物车

### 2.1 用户看到的完整流程

```
用户问："给布丁做一份这周的鲜粮计划"

       ↓

AI 输出：
┌─────────────────────────────────────────┐
│  布丁本周鲜粮计划                         │
│                                          │
│  📋 食材采购清单                           │
│  ┌──────────┬──────┬────────┬────────┐  │
│  │ 食材      │ 分量  │ 盒马    │ 永辉    │  │
│  ├──────────┼──────┼────────┼────────┤  │
│  │ 鸡胸肉    │ 500g │ ¥12.8  │ ¥11.5  │  │
│  │ 南瓜      │ 300g │ ¥4.5   │ ¥5.2   │  │
│  │ 糙米      │ 200g │ ¥3.2   │ ¥2.8   │  │
│  │ 蛋壳粉    │ 10g  │ —      │ —      │  │
│  └──────────┴──────┴────────┴────────┘  │
│                                          │
│  💡 AI建议：鸡胸肉和糙米选永辉更划算      │
│     南瓜选盒马更新鲜。总计约 ¥19.5。      │
│                                          │
│  [ 🛒 一键加入盒马购物车 ]                 │
│  [ 🛒 一键加入永辉购物车 ]                 │
│  [ 📦 智能分单：最优组合下单 ]             │
└─────────────────────────────────────────┘
```

**这个体验的核心差异：**

用户不需要打开 3 个 App、不需要自己搜商品、不需要对比价格、不需要一个一个加购物车。AI 食谱 → 比价表格 → 一键跳转购买。三步完成从"想做什么"到"食材到家"的闭环。

### 2.2 智能分单：AI 自动决定在哪买最划算

更进一步——AI 甚至可以帮用户"分单"：

- 鸡胸肉在永辉便宜 + 南瓜在盒马新鲜 = AI 建议分两单买，总成本 ¥19.5
- 如果你选"全部在盒马买"，AI 也可以一键帮你全部加进盒马购物车，总成本 ¥20.5
- 选哪个，你自己权衡。AI 只帮你算清楚，不替你做决定

### 2.3 三种购买模式，用户自己选

| 模式 | 适用场景 | 操作 |
|------|------|------|
| **单平台一键购** | 用户只装了盒马，懒得比价 | 所有食材一键跳转盒马小程序，自动填入购物车 |
| **AI 智能分单** | 追求性价比的用户 | AI 根据实时价格 + 新鲜度评分，把采购清单拆到不同平台，分别跳转下单 |
| **定期自动购** | 订阅了周度计划的用户 | 每周固定时间 AI 自动对比价格 → 推送确认通知 → 一键确认下单 |

## 3. 怎么实现？纯技术方案，不碰供应链

关键在于——**我们不需要和盒马/永辉签商务合作协议才能做这件事。** 技术上完全可以通过公开的小程序跳转和 Deeplink 实现。

### 3.1 实现方式：小程序跳转 + 搜索参数

盒马、永辉、叮咚买菜、美团闪购——这些平台都有自己的微信小程序或 App，而且支持带有搜索关键词的跳转：

| 平台 | 接入方式 | 能力 |
|------|------|------|
| **盒马** | 微信小程序路径跳转 + 搜索参数 | 跳转到盒马小程序，传递搜索关键词，用户看到搜索结果页 |
| **永辉生活** | 微信小程序跳转 + 搜索参数 | 同上 |
| **叮咚买菜** | 微信小程序跳转 | 同上 |
| **美团闪购** | 美团小程序跳转 | 搜索关键词 + 附近门店筛选 |
| **京东到家** | 京东小程序跳转 | 搜索关键词 + 附近门店筛选 |

**具体流程**：

1. AI 生成食材清单：鸡胸肉 500g、南瓜 300g、糙米 200g
2. 点"一键加入盒马购物车" → 系统拼接跳转 URL：`hemamini://search?keyword=鸡胸肉`
3. 用户跳到盒马小程序，看到鸡胸肉的搜索结果
4. 用户自己挑选下单（这一步在盒马内完成，我们不介入）

### 3.2 比价数据的来源

比价功能需要实时商品价格数据。几个可行的数据来源：

| 来源 | 覆盖 | 成本 | 时效性 |
|------|:---:|:---:|:---:|
| **平台公开 API**（如有） | 部分平台有开放的商品搜索 API | 低 | 实时 |
| **定期抓取 + 缓存** | 对热门食材（鸡胸肉/南瓜/胡萝卜等 50-100 个高频 SKU）做轻度价格采集和缓存 | 低 | 小时级 |
| **用户众包反馈** | "你买到的鸡胸肉在盒马多少钱？" → 用户自愿反馈，帮助优化比价数据 | 零 | 用户驱动 |
| **品牌合作后获取** | 如果能和盒马/永辉建立正式合作，直接接入商品价格 API | 零（合作互惠）| 实时 |

**MVP 阶段先做最简单的方案**：定期缓存 50-100 个高频宠物食材的价格，AI 比价基于缓存数据 + "价格可能波动，以平台实时价格为准"的免责声明。用户体验已经大幅好于"自己一个一个搜"。品牌合作后的实时价格接入是锦上添花。

### 3.3 用户端的极致简化

在宠食记 App 内，用户看到的是：

1. **比价结果页**：一张表格，列出每项食材在各平台的价格和配送时效
2. **一键跳转按钮**：点了就跳到对应平台的小程序，搜索词已经帮你填好了
3. **历史购买记录**：这周买了什么、花了多少钱、在哪买的——用于 AI 优化下次推荐

用户的支付、配送、售后全部在盒马/永辉/叮咚买菜内完成。我们不碰钱、不碰货、不碰售后。我们是帮用户做"购买决策"的那一层。

## 4. 分阶段落地路径

### 阶段一（MVP · 0-3 个月）：单平台跳转，验证"用户会点"

**做法**：
- AI 食谱输出时，食材清单下方有一个"🛒 在盒马购买这些食材"按钮
- 点击后跳转盒马小程序，搜索关键词自动填入
- 暂不做比价，只做跳转

**验证假设**：用户会从 AI 食谱页面点击购买按钮，并完成至少一次购买。

**技术实现**：微信小程序跳转 API（`wx.navigateToMiniProgram`）或 App Deeplink。不需要任何商务合作。

**成本**：几乎为零。开发工作量约 1-2 天。

### 阶段二（种子期 · 3-6 个月）：多平台比价 + 智能推荐

**前提**：阶段一验证通过——每周有稳定比例的购买跳转。

**做法**：
- 接入盒马 + 永辉 + 叮咚买菜三个平台
- 建立 50-100 个高频宠物食材的价格缓存，AI 比价
- 用户看到比价结果，自主选择从哪个平台买
- 启动和盒马/永辉的商务接触，探讨正式合作可能

**验证假设**：多平台比价能显著提升用户购买转化率（相比单平台跳转）。

### 阶段三（天使期 · 6-12 个月）：合作整合 + 购物车直填

**前提**：阶段二验证通过 + 至少和一家平台建立正式合作。

**做法**：
- 合作平台提供商品搜索 + 购物车 API 接入
- 用户点击"一键购买"后，食材直接填入该平台的购物车（不是跳到搜索页，是直接到购物车结算页）
- AI 智能分单：自动判断每项食材从哪个平台买最划算
- 订阅用户支持"每周自动采购"：AI 自动比价 → 推送确认 → 一键下单

**这个阶段的体验差距**：阶段一是"AI 帮你搜"，阶段三是"AI 帮你买"。

## 5. 和盒马/永辉谈合作的商业逻辑

和这些平台谈合作，不是去"求资源"，而是去"给价值"。我们需要清楚宠食记能给他们带来什么。

### 5.1 我们给盒马/永辉带来什么？

| 价值 | 具体说明 |
|------|------|
| **高 intent 转化流量** | 传统电商流量是"用户在盒马 App 里搜鸡胸肉"——他本来就要买。宠食记的流量是"AI 告诉用户你需要买鸡胸肉"——他本来可能没打算买，但 AI 让他产生了购买需求。这个流量的增量价值更高 |
| **精准需求预测** | 我们知道用户下周需要什么食材，因为 AI 已经生成了下周的饮食计划。盒马/永辉可以利用这个数据做需求预测和库存优化 |
| **新的增长场景** | "宠物鲜粮食材采购"是一个新的消费场景。盒马/永辉可能从来没有想过"有一群用户会每周固定时间来买宠物吃的鸡胸肉和南瓜"。这个场景可以帮他们开拓一个新的用户分层 |
| **零获客成本的分发渠道** | 宠食记的每一个食谱都是一条商品推荐。用户不需要盒马投广告——他们是带着"AI 明确说我要买这个"的意图来的 |

### 5.2 我们希望盒马/永辉提供什么？

| 需求 | 阶段 | 说明 |
|------|:---:|------|
| 允许小程序/App 跳转 | 短期 | 不需要合作，技术上已可行 |
| 商品价格 API 接入 | 中期 | 实现实时比价 |
| 购物车直填 API | 中长期 | 跳到购物车而非搜索页 |
| 联合会员权益 | 中长期 | 宠食记订阅用户享受盒马专属折扣 |
| 分佣或引流费用 | 中长期 | 按成交额 CPS 分佣 |

### 5.3 谈判策略

先做出量，再谈合作。不是"求你合作让我活下去"，而是"我已经帮你带来这么多成交了，我们是不是可以正式合作一下？"

具体来说：
1. **先用跳转把量跑起来**（阶段一不需要合作）。积累数据：每周有多少人通过宠食记跳转到盒马、实际购买转化预估是多少
2. **带着数据去谈**（阶段二）。"过去三个月，我们为盒马带来了 X 万次高 intent 搜索跳转，预估转化成交 Y 万元。我们可以把这个数据翻倍——如果你们愿意开放商品 API 的话"
3. **谈合作不是谈施舍，是谈增量**。宠物鲜粮食材采购是一个盒马/永辉自己还没有做、也不需要做（因为宠食记帮他们做了上游）的新场景

## 6. 电商之外的商超联动价值

和盒马/永辉的合作不仅仅是"卖食材"。

### 6.1 联合营销

- "宠物鲜粮周"活动：宠食记 + 盒马联合推出，盒马提供鸡胸肉/南瓜/胡萝卜优惠，宠食记提供专属食谱
- 用户订阅宠食记年度会员，赠送盒马 X 会员体验月卡（反之亦然）
- 线下联动：盒马门店设"宠物鲜粮食材角"，扫码就能在宠食记获取对应食谱

### 6.2 内容共创

- 宠食记在小红书发"盒马买食材做宠物鲜粮"系列内容，自然带出盒马的食材新鲜度和配送体验
- 盒马在自有内容渠道（盒区生活、盒马社群）推宠食记食谱，互相导流

### 6.3 数据合作（长期）

- 用户的食材购买行为数据 → 反馈给 AI："推荐了 A 食材但用户实际买了 B"→ 优化推荐模型
- 区域性食材偏好数据 → 帮助盒马/永辉做区域性选品

## 7. 法律合规

### 7.1 跳转盒马/永辉小程序是否合规？

**结论：完全合规。** 微信小程序跳转 API 是微信官方公开能力，不涉及任何侵权行为。跳转后用户在盒马/永辉小程序内完成购买，我们在法律上只是"信息导流"，不参与交易环节。

### 7.2 比价数据展示是否合规？

**结论：合规，但需注意几点。** 展示价格对比不构成侵权（这是公开信息）。需要注意：
- 标注数据来源（"价格来自盒马APP/永辉生活APP"）
- 标注数据时效性（"价格为查询时价格，以平台实时显示为准"）
- 不做虚假比价（不能捏造一个不存在的低价来诱导用户购买）

---

## 8. 整个策略一句话总结

**宠食记不做电商，做"购买决策层"。**

AI 食谱生成食材清单 → AI 帮你对比盒马/永辉/叮咚买菜的价格 → 一键跳转到你选的平台完成购买 → 食材到家之后 AI 提醒你开始做。

供应链、仓储、物流、支付、售后——全部由盒马/永辉们承担。我们只做他们不做的、AI 擅长做的：**帮你决定买什么、去哪买、怎么买最划算。**
