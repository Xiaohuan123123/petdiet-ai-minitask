## 开篇：我们要回答的问题

宠食记的 AI 给用户生成了一份鲜粮食谱——"鸡胸肉 80g、南瓜 60g、糙米 25g、蛋壳粉 1g"。然后呢？用户要自己打开购物 App，一个一个搜、一个一个加购物车。食谱的精准到了克，但购买的体验还停在"自己搞定"。

我们要解决的，就是这一段。

但有一个问题必须先想清楚：**宠食记要不要自己做电商？**

我们的答案是：**初期不追求做电商。我们做的是"AI 购买决策层"——AI 帮你决定买什么、告诉你去哪买最划算、帮你一键跳转完成购买。但库存、冷链、配送、售后，交给已经在做这些事的平台。**

这个定位决定了我们的策略：不是绑定某一家商超或平台，而是**连接一切能买到宠物食材的渠道，AI 帮用户做出最优选择。**

## 1. 为什么是"决策层"而不是"自建电商"？

一句话：**初期建电商太重，但完全不做购买链路又太轻。"决策层"是二者之间的最优解。**

对比一下三条路：

| 路径 | 做法 | 优势 | 劣势 |
|------|------|------|------|
| **自建电商** | 自己选品、入库、发货、售后 | 体验最闭环，收入空间最大 | 极重。供应链/仓储/物流/售后——哪一项都是巨坑。不适合初创期 |
| **纯 AI 工具** | 只出食谱，不给购买链接 | 最轻，零风险 | 用户要自己解决"去哪买"，体验断裂。食谱做到一半就断了 |
| **AI 决策层** ⬅ 我们的路 | AI 出食谱 → AI 比价 → AI 推荐最优渠道 → 一键跳转购买 | 轻资产，体验完整，不站队任何平台 | 收入靠 CPS 分佣，短期不高；需要多平台适配 |

**"决策层"不是"不做电商"的退而求其次。它是宠食记在这种品类结构下最合理的选择。**

为什么？因为宠食记推荐的商品，天然分散在多个不同类型的渠道：

- 一袋狗粮 → 淘宝/京东最合适（品类全、价格透明、可以比价）
- 500g 新鲜鸡胸肉 → 盒马/永辉/叮咚买菜最合适（生鲜、散称、30分钟达）
- 一瓶宠物益生菌 → 宠物垂直电商最合适（宠物专用、有正品保障）
- 用户的即时需求（"现在就要"） → 美团闪购/京东到家最合适

**没有一家平台能同时覆盖所有这些品类和场景。** 所以我们不做电商——我们做连接所有这些渠道的"决策层"。

## 2. 覆盖全渠道：不是只接盒马，是所有能买到宠物食材的地方

### 2.1 不同食材，天然适合不同平台

一张表说清楚：

| 食材类型 | 典型商品 | 最适合的平台 | 为什么 |
|------|------|------|------|
| **生鲜食材** | 鸡胸肉、南瓜、胡萝卜、蓝莓 | **盒马 / 永辉 / 叮咚买菜** | 散称零售、可只买几十克、线下门店覆盖、生鲜冷链成熟 |
| **标品干粮/湿粮** | 狗粮、猫粮、罐头 | **淘宝 / 京东** | 品类最全、品牌旗舰店、价格透明可全网比价、评价体系成熟 |
| **宠物专用补剂** | 钙粉、鱼油、益生菌 | **淘宝 / 京东宠物频道 / 宠物垂直电商** | 宠物专用品类，需要品牌授权和正品保障 |
| **即时性需求** | "现在就要"的食材 | **美团闪购 / 京东到家 / 饿了么** | 覆盖附近商超+药店，30分钟-1小时送达 |
| **本地化供给** | 鲜粮食材包、自制工具 | **本地宠物店 / 社区生鲜店** | 线下体验 + 社群关系，天然适合做熟客生意 |

**结论：不做判断"哪个平台更好"，而是告诉用户"你想要的东西，在哪些平台能买到，分别多少钱，哪个最划算"。AI 的价值是比价和推荐，不是替用户选平台。**

### 2.3 三种购买模式

| 模式 | 适用场景 | 操作 |
|------|------|------|
| **单平台一键购** | 用户不想折腾，一个平台搞定 | AI 推荐综合最优平台，所有食材一键跳转过去 |
| **AI 智能分单** | 追求性价比的用户 | AI 根据比价自动拆分——生鲜去哪买、标品去哪买，分别跳转。总成本最低 |
| **定期自动购** | 订阅了周度计划的用户 | 每周固定时间 AI 对比所有平台价格 → 推送确认通知 → 分别跳转下单 |

## 3. 怎么实现？纯技术方案，不需要商务合作就能起步

### 3.1 小程序跳转 + Deeplink 接入

所有这些平台都可以通过标准技术实现跳转，**不需要事先签商务合作协议**：

| 平台 | 接入方式 | 适合买什么 |
|------|------|------|
| **淘宝** | 淘宝客链接跳转 / 手淘 Deeplink | 狗粮猫粮、补剂、宠物用品 |
| **京东** | 京东联盟链接 / 京东 App Deeplink | 品牌旗舰店狗粮、宠物专用品 |
| **盒马** | 微信小程序跳转 + 搜索参数 | 生鲜食材（鸡胸肉、南瓜、蔬菜） |
| **永辉生活** | 微信小程序跳转 + 搜索参数 | 生鲜食材、散称零售 |
| **叮咚买菜** | 微信小程序跳转 | 生鲜食材 30 分钟达 |
| **美团闪购** | 美团小程序跳转 + 附近门店筛选 | 即时需求、附近商超 |
| **京东到家** | 京东小程序跳转 + 附近门店筛选 | 即时需求、附近商超 |
| **拼多多** | 多多进宝链接 | 价格敏感型用户、囤货 |

**这些跳转不需要任何一家平台的许可。** 淘宝客/京东联盟是平台官方推广渠道，微信小程序跳转是微信公开 API。

### 3.2 比价数据的实现

| 阶段 | 方案 | 覆盖 |
|------|------|:---:|
| **MVP** | 定期缓存 50-100 个高频食材的价格数据 + AI 推荐时标注"价格为预估，以平台实时价格为准" | 覆盖 80% 的常见宠物食材 |
| **种子期** | 接入平台公开商品搜索 API（如有），扩大缓存范围 | 覆盖 90%+ |
| **合作期** | 与部分平台建立正式合作后，接入实时价格和购物车 API | 实时比价 + 直填购物车 |

**MVP 阶段不需要实时价格。** 用户看到"鸡胸肉在盒马约 ¥12.8，在永辉约 ¥11.5"已经比"自己打开四个 App 分别搜"好了太多。价格浮动免责声明一句话就能覆盖。

## 4. 不同平台的合作逻辑

### 4.1 我们能给这些平台带来什么？

不管是盒马、永辉，还是淘宝、京东、美团——宠食记和它们的关系不是竞争，而是**上游导流**：

| 我们给平台的价值 | 具体说明 |
|------|------|
| **高 intent 增量流量** | 传统流量是"用户打开 App 搜商品"。宠食记的流量是"AI 告诉用户你需要买这个"——用户本来没打算买，AI 创造了他的购买需求。这个增量是平台自己做不到的 |
| **精准需求预测** | AI 已经生成了用户下周的饮食计划——我们知道他下周需要什么。这个预测数据对平台的供应链和库存管理有价值 |
| **新消费场景** | "给宠物做鲜粮"是一个新的消费理由。用户在盒马买鸡胸肉不是因为"今晚我要吃"，而是"我家狗这周要吃"——开拓了一个全新的增量场景 |
| **零成本的商品分发渠道** | 宠食记每一份食谱都是一张精准的商品推荐单。平台不需要为这些曝光付费 |

### 4.2 我们希望平台提供什么？

| 需求 | 优先级 | 说明 |
|------|:---:|------|
| 允许标准跳转 | — | 不需要合作，技术上已可行 |
| 商品价格 API | 🟡 中期 | 实现实时比价，提升 AI 推荐准确度 |
| 购物车直填 API | 🟡 中长期 | 跳过去直接到购物车结算页，不是搜索页 |
| CPS 分佣 | 🟢 短期 | 淘宝客/京东联盟/多多进宝已有，美团/盒马如有推广联盟也可接入 |
| 联合会员 | 🔵 长期 | 宠食记会员享平台专属折扣，双向导流 |

### 4.3 谈判策略：先跑量，再谈合作

不要先谈合作再做事。先用标准跳转把量跑起来，三个月后带着数据去谈：

"过去三个月，宠食记为你们带来了 X 万次精准商品搜索跳转，预估成交转化 Y 万元。我们可以把这个数据翻几倍——如果你们愿意开放商品价格 API 的话。"

这不是"求合作"，是"我们已经在帮你做生意了，要不要一起做得更好？"

## 5. 不同场景下的平台适配策略

不同的用户需求，AI 推荐不同的平台组合：

| 用户场景 | AI 推荐策略 | 典型组合 |
|------|------|------|
| **"我要给狗做本周的鲜粮"** | 生鲜食材优先推荐即时零售 | 盒马/永辉（生鲜）+ 淘宝（补剂） |
| **"我想换一款狗粮，不知道选哪个"** | 标品对比优先推荐电商平台 | 淘宝（比价）+ 京东（正品）+ 天猫旗舰店 |
| **"我现在就要，今晚就要做"** | 即时性优先 | 美团闪购/叮咚买菜（覆盖最广、最快） |
| **"我不急，囤货"** | 性价比优先 | 淘宝/拼多多（比价、满减、优惠券） |
| **"我不知道去哪买最好"** | AI 智能分单 | 跨平台最优组合 |

**核心原则：AI 不为任何平台站台。AI 只回答一个问题——"对于你现在需要的这些东西，去哪买最划算。"**

## 6. 电商之外的联动价值：内容 + 场景 + 数据

### 6.1 内容联合

- **小红书**：在宠食记账号发布"用 AI 出食谱 → 在淘宝/京东/盒马买食材 → 做出成品"的完整内容。每条内容自然带出购买平台的便利性（如"盒马 30 分钟就送到了"或者"京东自营次日达太方便了"）
- **平台侧**：如果和某家平台建立深度合作，可以在该平台的社区内容（如盒区生活、淘宝逛逛）同步推宠食记食谱，互相导流

### 6.2 数据闭环（长期）

- 用户的实际购买行为 → 反馈给 AI（"推荐了 A 平台的鸡胸肉，用户实际在 B 平台买了"）→ 优化推荐模型
- 区域性食材偏好数据 → 帮助合作平台做区域选品优化

## 7. 法律合规

| 风险点 | 判断 | 说明 |
|------|:---:|------|
| AI 食谱版权 | ✅ 无风险 | 食谱配方不受版权保护 |
| 淘宝客/京东联盟跳转 | ✅ 合规 | 平台官方推广渠道，已存在十几年 |
| 小程序跳转（盒马/永辉/美团等） | ✅ 合规 | 微信公开 API |
| 比价数据展示 | ⚠️ 注意标注 | 标注数据来源和时效性即可 |
| 食品安全连带责任 | ✅ 无风险 | 不生产、不经销，提供免责声明 |

## 8. 分阶段落地路径

### 阶段一（MVP · 0-3 月）：单渠道跳转验证

- AI 食谱附带 **淘宝/京东** 商品链接（干粮标品）+ **美团/盒马** 小程序跳转（生鲜食材）
- 不做比价，只做跳转
- 验证：用户会从 AI 食谱页面点击购买链接吗？更倾向于从哪个平台买？

### 阶段二（种子期 · 3-6 月）：多平台接入 + AI 比价

- 接入淘宝、京东、盒马、永辉、叮咚买菜、美团闪购六家平台的跳转
- 建立 50-100 个高频食材价格缓存，AI 输出比价表
- 启动和 1-2 家响应最快的平台深度接触

### 阶段三（天使期 · 6-12 月）：智能分单 + 购物车直填

- 合作平台开放购物车 API，跳转直接到结算页（不是搜索页）
- AI 智能分单——生鲜去哪买、标品去哪买、自动拆单
- 订阅用户支持"每周自动比价采购"

---

## 9. 一句话总结

**宠食记初期不做电商。我们做"AI 购买决策层"。**

AI 根据食谱生成采购清单 → AI 在所有可购渠道中比价（淘宝/京东/盒马/永辉/叮咚/美团）→ AI 推荐最优平台组合 → 一键跳转完成购买 → 食材到家。

供应链、冷链、仓储、物流、支付、售后——全部由合作平台承担。我们只做一件事：**帮用户决定买什么、去哪买、怎么买最划算。**

这不是不做电商的退而求其次。这是宠食记在这种品类结构下——标品+生鲜+补剂，散落在完全不同的渠道类型中——最合理的商业选择。
