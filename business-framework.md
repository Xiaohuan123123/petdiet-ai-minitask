# AI 宠物健康-饮食管理平台 · 商业方案框架

> 定位：不做 AI 兽医，做「兽医治疗前后的 AI 健康与饮食管理助手」
> 核心逻辑：把宠物健康问题从「单次问诊」变成「全周期管理」，AI 负责可规模化的个性化管理，兽医负责诊断和治疗

---

## 一、机会判断

### 1.1 为什么现在做

| 驱动因素 | 具体表现 |
|---|---|
| 宠物市场持续增长 | 2025 年中国城镇宠物（犬猫）数量达 **1.26 亿只**（犬 5343 万只、猫 7289 万只），城镇宠主达 **7874 万人**；2025 年市场规模 **3126 亿元**，预计 2028 年突破 **4000 亿元** |
| 宠物医疗支出刚性 | 医疗消费占宠物消费市场的 **27.6%**；看病贵是 80 后/80 前宠主最关注的痛点（占比超 50%） |
| 医疗后市场空白 | 兽医诊断后，主人往往不知道康复期怎么喂、怎么护理，信息碎片化 |
| AI 能力成熟 | 大模型能处理多模态输入（症状描述、化验单、食物照片），生成结构化、个性化的管理方案 |
| 用户付费意愿强 | 宠物是情感消费，**76.6%** 宠主购买宠物时最关注健康状况；**74.5%** 宠主将每月养宠消费控制在月收入 10% 以内，健康-饮食管理订阅（29 元/月）处于该预算带内 |

### 1.2 赛道核心矛盾

- **需求侧**：宠物主希望获得持续、专业、个性化的健康-饮食指导
- **供给侧**：兽医时间有限，营养师咨询贵且不持续，网络信息碎片化不可信
- **AI 切入点**：不替代兽医诊断，而是放大兽医价值，覆盖诊断前信息整理和诊断后持续管理

## 二、产品定位与价值主张

### 2.1 一句话定位

**"宠物的 AI 私人饮食健康管家，从日常喂养到病后康复，根据宠物状态动态调整。"**

### 2.2 完整价值主张

为宠物主提供覆盖宠物全生命周期的「动态饮食管理 + 健康辅助管理」服务：

- **日常期**：根据宠物品种、年龄、体重、活动量、季节变化，生成个性化日常饮食方案
- **变化期**：当宠物体重波动、精神状态变化、便便异常、过敏发作时，及时调整饮食
- **就医前**：帮你把宠物的症状、饮食、排便等情况整理成兽医能快速理解的报告
- **就医中**：记录诊断结果、用药方案、医嘱，形成结构化病历
- **就医后**：基于诊断生成个性化康复饮食方案和护理计划
- **全周期**：持续追踪体重、饮食、行为、排便，让饮食方案随宠物状态动态进化

### 2.3 与竞品和人类的边界

| 角色 | 做什么 | 不做什么 |
|---|---|---|
| **AI 助手** | 信息整理、饮食方案、康复管理、健康监测、就医引导 | 疾病诊断、开具处方、替代兽医 |
| **兽医** | 诊断、治疗、开药、手术 | 长期日常饮食管理、24 小时随访 |
| **宠物医院** | 线下诊疗、检查、手术 | 患者出院后持续管理 |

**核心原则**：AI 永远作为辅助，诊断和治疗决策权在人类兽医手中。

## 三、目标用户与分层

### 3.1 核心用户画像

| 用户类型 | 特征 | 核心痛点 | 付费动机 |
|---|---|---|---|
| **新手宠物主** | 养宠 0–2 年，第一次遇到宠物生病；00 后、90 后合计占比近 **70%** | 不知道什么情况下要就医，就医前不知道怎么描述 | 避免误诊、减少焦虑 |
| **慢性病宠物主** | 宠物有肾病、糖尿病、肥胖、心脏病等需长期管理；6 岁以上猫不健康占比超 **50%**，9 岁以上犬不健康占比 **53.4%** | 需要长期饮食控制和复查跟踪，兽医没时间逐月指导 | 控制病情、延长宠物寿命 |
| **高消费宠物主** | 愿意为宠物花大钱，追求科学养宠；高消费人群中 90 后占 **50.1%**，00 后占 **27.3%** | 希望获得个性化、专业化的健康管理 | 省时、省心、专业 |
| **多宠家庭** | 养 2 只以上宠物 | 管理多只宠物的饮食、健康、病历很混乱 | 统一管理、提高效率 |

### 3.2 早期付费决策者

1. **慢性病宠物主**：需求最刚性，付费意愿最强，复购率最高
2. **术后/病后宠物主**：短期高 intent，愿意为康复方案一次性付费
3. **高消费新手宠物主**：焦虑感强，愿意为"科学养宠"付费

## 四、用户旅程与功能矩阵

### 4.1 用户旅程地图

```
阶段 0：日常养护
├─ 记录体重、饮食、排便、运动
├─ 获取日常营养建议
└─ 接收疫苗/驱虫提醒

阶段 1：发现异常
├─ 拍照/描述症状
├─ AI 初步整理症状时间线
└─ 判断是否需要就医 + 就医紧迫度

阶段 2：就医准备
├─ 生成就医摘要（症状 + 饮食 + 生活习惯）
├─ 推荐检查项目和问诊问题
└─ 预约附近宠物医院

阶段 3：诊断与治疗
├─ 记录诊断结果、化验单、用药方案
├─ 解读医嘱和药品说明
└─ 设置用药提醒

阶段 4：康复管理
├─ 生成康复期饮食方案
├─ 制定护理计划（喂食、活动、观察指标）
├─ 每日打卡跟踪
└─ 异常时提醒就医

阶段 5：长期健康管理
├─ 慢病饮食方案动态调整
├─ 体重/指标趋势分析
├─ 复查提醒
└─ 兽医随访协助
```

### 4.2 功能-变现矩阵

| 用户阶段 | 免费/获客功能 | 核心付费功能 | 高价值/企业功能 |
|---|---|---|---|
| **日常养护** | 宠物档案、体重记录、基础饮食建议、疫苗驱虫提醒 | 多宠物管理、高级营养方案、成分解码 | 宠物食品品牌数据服务 |
| **异常发现** | 症状自评、AI 症状整理、就医紧迫度判断 | 24h 智能健康顾问、专家问诊预约 | 宠物医院导流分成 |
| **就医准备** | 就医摘要生成、问诊问题清单 | 医院推荐 + 预约、病历结构化存储 | 医院 SaaS 患者管理系统 |
| **康复治疗** | 用药提醒、基础护理建议 | **康复饮食方案**、康复期跟踪、复查提醒 | 宠物医院康复管理合作 |
| **慢病管理** | 慢病知识库、症状日记 | **慢病饮食订阅**、动态方案调整、趋势报告 | 保险预核保、药企患者项目 |

### 4.3 功能优先级判断标准

每个功能用四个维度打分：

| 维度 | 权重 | 说明 |
|---|---|---|
| **AI 原生性** | 30% | 没有 AI 是否难以规模化实现 |
| **用户痛点强度** | 25% | 用户是否愿意为这个功能付费 |
| **法律/信任风险** | 20% | 是否涉及诊断、用药、医疗责任 |
| **商业化速度** | 15% | 能否在 3 个月内产生收入 |
| **数据沉淀价值** | 10% | 是否有助于建立长期护城河 |

## 五、商业模式设计

### 5.1 收入模式组合

```
┌─────────────────────────────────────────────────────┐
│                  商业模式金字塔                      │
├─────────────────────────────────────────────────────┤
│  顶层：企业级服务（医院 SaaS / 品牌数据 / 保险合作）   │
│         高客单价，长周期，强护城河                    │
├─────────────────────────────────────────────────────┤
│  中层：订阅会员（慢病管理 / 多宠家庭 / 高级功能）      │
│         稳定现金流，高 LTV                            │
├─────────────────────────────────────────────────────┤
│  底层：单次付费（康复饮食方案 / 就医前报告 / 营养评估） │
│         低门槛，高 intent，快速变现                   │
├─────────────────────────────────────────────────────┤
│  基础层：免费工具（症状整理 / 成分解码 / 用药提醒）    │
│         获客入口，建立信任                            │
└─────────────────────────────────────────────────────┘
```

### 5.2 C 端定价设计

| 版本 | 价格 | 权益 |
|---|---|---|
| **免费版** | 0 元 | 宠物档案、基础饮食建议、成分解码（每月 3 次）、用药提醒 |
| **单次版** | 9.9–29.9 元/次 | 康复饮食方案、就医前整理报告、高级营养评估 |
| **会员版** | 29–59 元/月 | 无限次方案生成、慢病管理、多宠物档案、趋势报告、专家问诊折扣 |
| **家庭版** | 99 元/月 | 最多 5 只宠物、全家共享、优先客服、医院预约绿色通道 |

*定价依据：74.5% 宠主将每月养宠消费控制在月收入 10% 以内；单只猫年均消费 2085 元、单只犬年均消费 3006 元。29 元/月订阅约占月均养宠支出的 10%～17%，处于用户可接受区间。*

### 5.3 B 端收入路径

| B 端客户 | 合作形式 | 收入来源 |
|---|---|---|
| **宠物医院** | 提供患者出院后管理工具、导流复诊 | SaaS 订阅、导流分成 |
| **宠物食品品牌** | 成分数据、用户画像、推荐位 | 数据服务、广告/导购佣金 |
| **宠物保险公司** | 健康数据预核保、理赔辅助 | 保单分成、数据服务费 |
| **宠物药企** | 慢病患者管理项目、用药依从性跟踪 | 项目服务费 |

## 六、数据飞轮与护城河

### 6.1 数据飞轮

```
用户使用免费工具
    ↓
积累宠物档案、症状、饮食、诊断数据
    ↓
AI 推荐质量提升
    ↓
用户付费转化率提升
    ↓
更多数据 → 更准的模型 → 更高用户价值
    ↓
B 端医院/品牌/保险愿意付费购买数据服务
```

### 6.2 护城河来源

1. **垂直数据壁垒**：宠物健康-饮食-诊断的关联数据，通用 AI 没有
2. **用户迁移成本**：档案、病历、历史方案越多，用户越不愿意换平台
3. **医院/品牌生态绑定**：B 端合作关系形成渠道壁垒
4. **信任品牌**：长期正确建议积累的专业信任，新进入者难以短期复制

## 七、风险框架与规避策略

### 7.1 核心风险

| 风险 | 等级 | 应对策略 |
|---|---|---|
| **医疗责任风险** | 🔴 高 | 明确免责声明，AI 仅作辅助建议，不诊断、不开处方；高风险建议强制引导就医 |
| **市场竞争/红海风险** | 🔴 高 | 避开美团/大众点评的本地导流和信息撮合赛道，聚焦"病后康复饮食+慢病管理"垂直纵深；用数据壁垒和订阅粘性建立差异化 |
| **AI 输出准确性** | 🟠 中 | 基于权威资料构建知识库，关键建议加医学校验层；建立用户反馈闭环 |
| **用户信任建立** | 🟠 中 | 与宠物医院/兽医合作背书；免费工具先建立信任，再引导付费 |
| **获客成本** | 🟠 中 | 从宠物医院、宠物社群、小红书等高信任渠道切入；内容获客；避免与平台级玩家正面竞价 |
| **大厂降维打击** | 🟠 中 | 美团/大众点评掌握本地流量和配送网络，但 unlikely 深耕重责任的宠物病后健康管理；项目应做"大厂不愿做、做不好"的垂直场景 |
| **竞品模仿** | 🟡 低-中 | 靠数据、订阅粘性和 B 端生态绑定构建壁垒，单点功能容易被抄 |

**红海竞争数据支撑**：详见 [`red-sea-competition-data.md`](./red-sea-competition-data.md)。关键数据包括：全国宠物医院 3.7 万+家、食品品牌 CR10 仅 26%～33.7%、小红书宠物搜索量同比 +41%、美团闪购 2025 年 GTV 预计 3400 亿元。

### 7.2 法律边界

- 永远不使用「诊断」「治疗」「处方」等医疗术语
- 所有健康建议标注"仅供参考，具体情况请咨询兽医"
- 对可能危及生命的症状，必须引导立即就医
- 不售卖药品，只做用药提醒和说明

## 八、短期可商业化功能推荐

### 8.1 推荐顺序

| 排名 | 功能 | 商业化形式 | 推荐理由 |
|---|---|---|---|
| **🥇 第一** | **AI 宠物全周期饮食管理订阅** | 月费 29–59 元 | 需求高频、订阅属性强、AI 原生性强、覆盖日常+病后全场景 |
| **🥈 第二** | **病后/术后康复饮食方案** | 单次付费 19.9–39.9 元 | 高 intent、低法律风险、与医疗链条自然衔接 |
| **🥉 第三** | **动态饮食调整服务** | 会员权益 / 单次 9.9 元 | 基于体重/便便/状态变化及时调整，增强粘性 |
| 第四 | 就医前症状整理报告 | 单次付费 9.9–19.9 元 | 使用门槛低、获客效果好、可作为付费转化入口 |
| 第五 | 宠物医院预约导流 | 导流佣金 | 变现快，但需要建立医院合作网络 |

### 8.2 为什么首推「AI 宠物全周期饮食管理订阅」

1. **覆盖场景最广**：既包括日常喂养，也包括病后康复、体重管理、过敏管理、换粮期等
2. **使用频率最高**：饮食是每天都要做的事，用户会高频打开
3. **订阅属性最强**：宠物状态持续变化，饮食方案需要动态调整，天然适合订阅
4. **数据护城河最深**：持续记录饮食、体重、便便、行为，模型越用越准
5. **向上销售路径清晰**：免费日常建议 → 订阅全周期管理 → 病后专项方案 → 医院合作服务

### 8.3 最小商业化闭环（3 个月内）

```
获客：免费日常饮食建议 / 小红书内容
    ↓
转化：用户发现宠物状态变化（体重波动/便便异常/换粮需求）
    ↓
付费：订阅月度饮食管理 或 购买单次动态调整方案
    ↓
留存：每日喂养打卡、体重记录、便便记录
    ↓
复购：订阅续费 + 病后专项方案 + 医院合作服务
```

### 9.1 三年目标

成为中国宠物主最信任的 **"AI 健康-饮食管理入口"**，连接宠物主、兽医、宠物食品品牌、保险公司，形成宠物健康生态的数据中枢。

```
Phase 1（0–6 个月）：单点验证

├─ 核心功能：医院合作、保险对接、品牌数据服务
├─ 目标：建立 B 端收入和数据护城河
└─ 商业模式：B2B2C 收入占比提升

Phase 4（2–3 年）：平台化
├─ 核心功能：开放 API、行业标准数据服务
├─ 目标：成为宠物健康数据基础设施
└─ 商业模式：数据服务 + 平台分成
```

## 十、关键假设与待验证问题

### 10.1 核心假设

1. **H1**：宠物主愿意为病后康复饮食方案付费
2. **H2**：AI 生成的康复饮食方案在专业性和个性化上显著优于免费网络信息
3. **H3**：用户愿意持续记录宠物健康数据，形成使用习惯
4. **H4**：宠物医院愿意合作，把 AI 工具作为患者管理延伸

**市场数据验证**：上述假设已被《2026 年中国宠物行业白皮书》部分验证：
- 2025 年城镇犬猫 1.26 亿只、宠主 7874 万人，用户基数大（验证市场天花板）
- 76.6% 宠主购买宠物时最关注健康状况，21.3% 犬主人、32.1% 猫主人认为宠物不健康（验证 H1 需求存在）
- 食品市场份额 53.7%、营养品渗透率 66.3% 且持续上升，说明宠主愿为饮食健康付费（验证 H1/H2）
- 90 后+00 后合计占比近 70%，高学历、理性消费，更易接受 AI 工具（验证 H3 用户基础）
- 医疗消费中"重预防、轻治疗"趋势明显，体检份额上升、诊疗/药品份额下降（验证 H4 合作空间）

详细数据索引见 [`market-data-library.md`](./market-data-library.md)。

### 10.2 2 周验证重点
