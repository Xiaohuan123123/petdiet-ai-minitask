## 0. 定位校准：为什么一切要围绕"吃什么"

你提的这一点非常关键。我们前面分析过很多痛点，但本质上用户最高频、最具体、最愿意为之付费的问题，都可以收敛到一个问题上：**我的宠物今天该吃什么？**

健康焦虑的背后，是"生病了该吃什么"。  
饮食决策困难的背后，是"日常该吃什么"。  
换粮/过敏/体重问题的背后，是"吃什么才对"。  
甚至用户搜索"猫软便怎么办"，最终答案往往也指向"该换什么粮、怎么喂"。

所以这一次的功能规划，我们不再把"健康"和"饮食"并列，而是把**健康信息作为输入，饮食方案作为输出**。医疗相关的功能——症状记录、病历管理、用药提醒——都是为了更好地回答"吃什么"这个问题。

这个定位有三个好处：
1. **更聚焦**：用户打开产品时，心里有一个明确的问题期待被回答
2. **更差异化**：大厂做健康做医疗做电商，但很少有一个产品把"吃什么"做透
3. **更易变现**：饮食方案天然可以衔接内容、服务、品牌合作，而不是只做一次性问诊

## 1. 功能主线：宠物"吃什么"的完整闭环

我们把"吃什么"拆成一条完整的问题链：

```
宠物现在是什么状态？
    ↓
不同年龄/品种/体重该吃什么？
    ↓
生病了该吃什么？怎么吃？
    ↓
日常喂现成的还是新鲜的？怎么配？
    ↓
怎么判断粮好不好？怎么买？
    ↓
吃了之后效果怎么样？要不要调整？
    ↓
不懂的时候去哪里学？怎么分享？
```

对应这条链，我们重新组织功能模块：

| 问题 | 功能模块 |
|---|---|
| 宠物现在是什么状态？ | 宠物基础健康信息收集 |
| 不同年龄/品种/体重该吃什么？ | 全生命周期饮食 + 日常饮食规划 |
| 生病了该吃什么？ | 生病周期饮食 + 药物服用规划 |
| 日常喂现成的还是新鲜的？ | 现成干粮配置 + 新鲜粮食配置 + 自制粮教程 |
| 怎么判断粮好不好？ | 饮食知识库 + 粮食鉴定/评测 |
| 吃了之后效果怎么样？ | 每日记录 + 动态调整 |
| 不懂的时候去哪里学？ | AI 科普 + RAG 知识库 |
| 怎么分享/裂变？ | 动物朋友圈 + 内容活动策划 |

## 2. 整合后的功能模块规划

### 2.1 宠物基础健康信息收集

**定位**：这是回答"吃什么"的第一步。没有健康信息，饮食方案就是猜。

**收集维度**：
- 基础档案：品种、年龄、性别、体重、绝育状态
- 健康状态：既往病史、过敏史、慢性病、当前用药、疫苗/驱虫记录
- 饮食现状：目前在吃什么粮、每天吃多少、零食情况、饮水情况
- 行为状态：运动量、精神状态、排便情况

**为什么这些都要收集？**

因为"吃什么"不是孤立的。一只 8 岁的肾病猫和一只 1 岁的健康猫，答案完全不同。一只每天遛 2 小时的狗和一只宅家的猫，热量需求也完全不同。甚至一只正在吃处方药的宠物，饮食方案还要考虑药物和食物的相互作用。

**MVP 范围**：
- 基础档案 + 健康状态 + 饮食现状
- 支持拍照上传化验单、疫苗本
- 支持多宠物（MVP 先 1～2 只）

---

### 2.2 宠物饮食规划（每日记录）

**定位**：这是产品最高频的使用场景。用户每天打开产品，核心问题就是"今天喂什么、喂多少"。

**功能点**：
- **每日饮食计划**：根据档案自动生成今日建议（粮种、克数、次数）
- **喂食记录**：记录实际吃了多少、剩余多少、是否吃完
- **饮水记录**：尤其对猫很重要，饮水不足是肾病的重要诱因
- **便便记录**：形态、颜色、频率，这是判断消化健康最直接的指标
- **体重记录**：每周一次，生成趋势曲线

**每日记录的价值**：

很多宠物主其实有记录的意愿，但不知道记录什么、记录有什么用。我们要让记录行为立刻产生反馈——比如记录完今天的喂食和便便，AI 会给出一句"今天状态不错"或者"最近 3 天软便，建议减少 10% 喂食量"。这种即时反馈是养成习惯的关键。

**MVP 范围**：
- 每日饮食计划
- 喂食记录
- 便便记录
- 体重记录

---

### 2.3 全生命周期饮食

**定位**：宠物在不同生命阶段，营养需求完全不同。饮食方案不能一成不变。

**生命周期划分**：
- 幼年犬猫（快速成长，需要高蛋白、高能量）
- 青年犬猫（维持体态，预防肥胖）
- 中年犬猫（关注代谢、关节、泌尿）
- 老年犬猫（低磷、低钠、易消化、关节保护）

**功能点**：
- 根据生日自动判断生命阶段
- 不同阶段自动切换饮食重点
- 阶段切换时提供过渡方案
- 老年宠物自动增加健康监测频率

**和商业化的关系**：

全生命周期饮食是订阅制的重要理由。用户不是为了一天两天的建议付费，而是为"宠物从 1 岁到 15 岁，方案一直跟着变化"付费。

---

### 2.4 生病周期饮食 + 药物服用规划

**定位**：这是产品最刚性的付费场景，也是健康信息服务于"吃什么"的核心体现。

**功能点**：
- **病种饮食方案库**：覆盖肠胃炎、皮肤病、泌尿系统问题、肾病、糖尿病、肥胖、术后恢复等
- **个性化方案生成**：输入诊断结果、医嘱、当前用药，生成 7～14 天饮食计划
- **药物服用规划**：记录用药时间、剂量，设置提醒，避免与饮食冲突
- **康复跟踪**：每日打卡饮食、便便、精神、体重，异常时预警
- **复查提醒**：根据病种自动提醒复查时间

**为什么药物规划也要做？**

因为很多药需要空腹吃、饭后吃、或者不能和某些食物同服。比如一些抗生素需要饭后服用以减少肠胃刺激，而有些药需要空腹。如果只给饮食方案不管用药时间，用户执行起来会混乱。

**边界**：
- 不诊断疾病，只基于用户输入的诊断结果做饮食管理
- 所有方案标注"请遵医嘱，必要时咨询兽医"
- 不直接推荐药品或改变用药方案

---

### 2.5 现成干粮配置（购买策略 + 投喂策略）

**定位**：解决用户"买什么粮、怎么喂"的问题。

**功能点**：
- **购买策略**：
  - 根据宠物档案推荐适配的干粮品牌和品类
  - 预算分级推荐（平价/中端/高端）
  - 适口性、消化率、成分透明度等多维度评分
  - 换粮建议（什么时候该换、换什么）
- **投喂策略**：
  - 每日喂食量计算
  - 喂食次数建议（幼年期多频次、成年期可定时）
  - 干湿粮搭配建议
  - 零食控制建议

**关于卖货的中立性平衡**

直接把卖货作为核心商业模式，确实会让我们变成电商平台，和大厂以及垂直宠物电商正面竞争，而且可能损害中立性。但考虑到用户决策链路——"知道吃什么"之后自然就是"怎么买"——我们决定在 P0 阶段就接入购买链路。

**边界设定**：
- 推荐逻辑保持中立：基于宠物适配度、成分、预算评分，而不是佣金高低
- 不压货、不自营：通过联盟链接、品牌小程序、线下门店导流完成交易
- 透明标注：明确区分"AI 推荐"和"合作推荐"
- 重点赚"方案"的钱，而不是"商品差价"：购买是服务的延伸，不是核心利润来源

这样既能闭环用户体验，又避免重资产投入和中立性崩盘。

### 2.6 新鲜粮食配置 + 自制粮食制作教程

**定位**：满足进阶用户的需求。越来越多的宠主想喂鲜粮或自制粮，但不知道配比、担心营养不均衡。

**功能点**：
- **新鲜粮食配置**：
  - 根据宠物需求推荐鲜粮品牌或配方
  - 评估鲜粮与干粮的搭配比例
  - 安全性和营养配比提醒
- **自制粮教程**：
  - 基础版：简单辅食、零食制作
  - 进阶版：完整自制主食配方（肉类、蔬菜、碳水、营养补充剂比例）
  - 根据宠物状态动态调整配方
  - 食材禁忌提醒（如猫不能吃什么、肾猫不能吃什么）

**为什么要做自制粮教程？**

白皮书数据显示，39.2% 的宠主想喂鲜粮是因为追求更健康，但 34.0% 拒绝鲜粮是因为"不方便"。这里的"不方便"包括购买、存储、喂食，也包括"不知道怎么做"。如果我们能降低自制粮的门槛，就能打开一个增量市场。

**风险提醒**：
- 自制粮必须强调营养配比，不能让用户随便做
- 需要明确标注"长期自制主食建议咨询宠物营养师"

---

### 2.7 饮食知识库搭建（AI 科普 + RAG）

**定位**：建立产品的专业信任感。用户不懂的时候，能在我们这里找到靠谱答案。

**功能点**：
- **结构化知识库**：
  - 犬猫营养学基础
  - 常见成分解读
  - 各病种饮食要点
  - 食材禁忌
  - 换粮/过渡指南
- **AI 问答**：
  - 基于 RAG（检索增强生成）技术，让 AI 的回答有据可依
  - 每个回答标注参考来源
  - 不确定的问题不瞎答，引导就医或咨询专业人士
- **科普内容**：
  - 短文、图文、短视频形式
  - 与小红书、抖音内容矩阵联动

**RAG 的价值**：

宠物健康领域最大的问题是信息鱼龙混杂。用 RAG 技术，我们可以让 AI 的回答基于权威资料（如兽医教材、行业白皮书、品牌成分数据），而不是凭空生成。这能显著提升用户信任度。

### 2.8 正版/好物粮食、玩具鉴定（得物路线）

**定位**：这是一个需要谨慎评估的功能。你提到的"得物路线"我理解是想做正品鉴定和好物评测，增强信任感和商业化。

**我们判断**：

这个功能**可以做，但要控制边界**，否则会变成重资产的电商/鉴定平台，偏离"吃什么"的核心定位。

**建议做法**：
- **轻量版（推荐）**：
  - 提供"真伪鉴别指南"：教用户如何通过包装、防伪码、批次号判断真伪
  - 提供"好物评测"：基于成分分析、用户反馈、第三方检测数据，给出中立评分
  - 不直接做鉴定交易，不承诺"保真"
- **不做**：
  - 自建鉴定团队（成本太高）
  - 直接卖货赚差价（和大厂竞争，损害中立性）
  - 玩具鉴定（偏离饮食核心）

**商业化路径**：
- 通过"好物榜单"和"成分评测"建立专业信任
- 未来可以接入品牌官方的正品查询接口
- 为品牌提供数据服务（成分关注度、用户偏好）

---

### 2.9 链接线上线下平台（商业化）

**定位**：P0 阶段先打通"线上饮食方案 → 线上购买"的最小闭环；P2 阶段再扩展到线下服务连接。

**P0 阶段（线上购买闭环）**：
- 根据饮食方案推荐适配粮，跳转至天猫/京东/品牌小程序完成购买
- 支持按预算、品类、品牌偏好筛选
- 记录购买历史，用于后续效果跟踪和复购提醒

**P2 阶段（线上线下融合）**：
- **线上 → 线下**：
  - 根据饮食方案推荐附近的宠物医院、宠物店
  - 复查提醒时推荐合作医院
  - 需要购买特定粮时推荐附近门店
- **线下 → 线上**：
  - 医院给用户开诊断后，推荐用户使用我们的康复饮食方案
  - 宠物店推荐用户使用我们的日常饮食规划
- **品牌合作**：
  - 与宠物食品品牌合作，提供成分数据、用户洞察
  - 为品牌提供"定制化饮食方案"联名服务

**商业化原则**：
- 用户体验优先，不能因为商业化推荐损害信任
- 推荐逻辑透明，标注"合作推荐"或"中立推荐"
- 数据共享需用户授权且匿名化

---

### 2.10 狗猫粮食鉴定 + 品牌合作

**定位**：这是商业化的一部分，但需要特别小心。

**建议边界**：
- **粮食鉴定**：轻量做，提供方法和工具，不做重人工鉴定
- **品牌合作**：选择有资质、有口碑的品牌，合作形式以内容和数据服务为主
- **避免**：
  - 为了佣金推荐不合适的产品
  - 让品牌合作影响 AI 建议的中立性
  - 直接做"独家代理"或"自营商品"

**更健康的合作方式**：
- 品牌赞助科普内容
- 品牌提供成分数据，我们用于 AI 推荐
- 品牌联合推出"某某病种饮食方案"（需兽医审核）

---

### 2.11 猫狗运动规划指导（宠物界的 Keep）

**定位**：运动是影响宠物健康的重要因素，尤其是肥胖、糖尿病、关节问题。但它是"吃什么"的辅助功能，不是核心。

**功能点**：
- 根据品种、年龄、体重、健康状况推荐运动量
- 提供运动计划（狗狗每日遛狗时长、猫咪室内活动建议）
- 结合体重变化调整运动建议
- 与饮食方案联动："运动量增加 → 热量需求增加 → 饮食方案调整"

**优先级**：
- MVP 不做
- P2 阶段作为会员增值功能
- 重点放在"饮食 + 运动"的联动，而不是做一个独立的宠物 Keep

---

### 2.12 动物朋友圈分享（垂类社交，活动策划）

**定位**：你提到这不是核心，但要做活动策划增加裂变和流量传播。我们同意这个判断。

**功能点**：
- **轻量分享**：生成"我家宠物的饮食报告""康复打卡海报"等可分享图片
- **话题活动**：
  - "21 天健康饮食打卡"
  - "我家毛孩子的今日餐盘"
  - "肾病猫饮食挑战"
- **社群入口**：引导用户加入微信群/小红书群，做慢病宠物主交流

**不做**：
- 独立的社交 Feed 流（成本高、留存差）
- 复杂的宠物交友功能

**裂变机制**：
- 打卡分享获得会员时长
- 邀请好友注册获得成分解码次数
- 老用户推荐新用户双方获益

## 3. 重新梳理后的功能架构

围绕"吃什么"，产品能力可以抽象为四层：

```
┌─────────────────────────────────────────────┐
│  用户触达层：让用户每天来、愿意分享          │
│  小程序/APP、每日记录、分享海报、社群活动    │
├─────────────────────────────────────────────────────────────┤
│  方案输出层：回答"吃什么"                    │
│  日常饮食、全生命周期饮食、生病周期饮食、    │
│  干粮配置、鲜粮/自制粮配置                   │
├─────────────────────────────────────────────────────────────┤
│  智能支撑层：让方案更准                      │
│  AI 饮食生成、动态调整引擎、知识库 RAG、     │
│  成分分析、症状自评                          │
├─────────────────────────────────────────────────────────────┤
│  数据基础层：让方案有依据                    │
│  宠物档案、健康记录、饮食记录、病历管理      │
└─────────────────────────────────────────────┘
```

## 4. MVP 范围（重新调整）

基于你提供的清单和"围绕吃什么"的定位，MVP 需要回答一个问题：**用户第一次使用产品时，能不能快速得到"我的宠物今天该吃什么"的答案？**

### MVP Must Have

1. **宠物基础健康信息收集**
   - 基础档案 + 健康状态 + 饮食现状
2. **宠物饮食规划（每日记录）**
   - 每日饮食建议 + 喂食记录 + 便便记录 + 体重记录
3. **生病周期饮食 + 药物服用规划**
   - 至少覆盖 3～5 个常见病种（肠胃炎、术后、皮肤病、泌尿系统）
   - 用药提醒
4. **现成粮食配置 + 自制鲜粮制作 AI 辅助**
   - 干粮购买策略 + 投喂策略
   - 鲜粮/自制粮配方推荐 + 制作教程
5. **粮食推荐和购买**
   - 基于宠物档案推荐适配粮
   - 接入购买链路（电商/品牌/线下门店）
6. **饮食知识库（AI 科普 + RAG）**
   - 基础问答 + 成分解读 + 常见饮食问题
7. **支付系统**
   - 支持单次购买病后方案、购买推荐粮

### MVP Should Have

1. **就医前症状整理报告**
   - 辅助用户和兽医沟通
2. **病历拍照上传**
3. **全生命周期饮食**
   - 根据年龄自动调整饮食重点
4. **动态饮食调整**
   - 根据记录反馈优化方案

### MVP Won't Have

1. **粮食鉴定/好物鉴定**（先以评测内容形式存在）
2. **链接线上线下平台**（需要建立合作关系，MVP 先聚焦线上购买）
3. **品牌深度合作**（信任未建立前不建议做）
4. **运动规划指导**（辅助功能，P2 再做）
5. **动物朋友圈**（先做分享海报，不做完整社交功能）
6. **AI 疾病诊断**（法律风险，不做）

**MVP 取舍说明**：

这次把"现成粮食配置""自制鲜粮制作""粮食推荐和购买"都放进了 P0，意味着 MVP 就要回答"吃什么、怎么买"的完整闭环。这会让 MVP  heavier，但也能更快验证商业闭环——用户不仅来获取建议，还可以直接完成购买转化。

需要承担的代价是：MVP 需要接入电商或品牌购买链路，产品复杂度和合规要求都会上升。建议在 P0 阶段先接入 1～2 个稳定的供应渠道（如天猫/京东联盟、品牌官方小程序），而不是自建电商。

## 5. 功能优先级排序（围绕"吃什么"主线）

我们用"对回答'吃什么'的贡献度"作为核心排序标准。

### P0：MVP 必须包含

| 功能 | 为什么 P0 |
|---|---|
| 宠物基础健康信息收集 | 没有它就无法个性化回答 |
| 宠物饮食规划（每日记录） | 最高频使用场景，养成用户习惯 |
| 生病周期饮食 + 药物规划 | 最刚性的付费场景 |
| 现成粮食配置 | 直接回答"买什么、喂多少" |
| 自制鲜粮制作 AI 辅助 | 满足进阶用户"怎么做"的需求 |
| 粮食推荐和购买 | 闭环"吃什么 → 怎么买" |
| 饮食知识库（RAG） | 建立信任，解释"为什么这样吃" |

### P1：让"吃什么"更精准、更可执行

| 功能 | 为什么 P1 |
|---|---|
| 全生命周期饮食 | 让"吃什么"随年龄变化 |
| 就医前症状整理 | 连接医疗场景，输入更准确 |
| 病历管理 | 让饮食方案有据可依 |
| 动态饮食调整 | 根据反馈优化方案 |

### P2：扩展"吃什么"的边界和商业化

| 功能 | 为什么 P2 |
|---|---|
| 粮食评测/好物推荐 | 内容获客 + 品牌合作基础 |
| 运动规划指导 | 饮食 + 运动联动 |
| 多宠家庭版 | 细分用户付费 |
| 动物朋友圈分享 | 裂变传播 |
| 线上线下链接 | 商业化放大 |
| 品牌深度合作 | 基于用户规模和信任后再做 |

### 不做：偏离"吃什么"核心或风险过高

| 功能 | 原因 |
|---|---|
| AI 疾病诊断 | 法律风险 |
| 直接卖货/自营电商 | 和大厂竞争，损害中立性 |
| 宠物社交 Feed 流 | 与核心价值无关 |
| 宠物出行/殡葬 | 低频且偏离主线 |

## 6. 商业化路径（围绕"吃什么"）

### 免费层：建立信任

- 宠物档案
- 基础每日饮食建议
- 基础饮食知识问答
- 有限次成分解码
- 症状自评

### 单次付费层：解决高 intent 场景

- 病后康复饮食方案：19.9～39.9 元
- 就医前症状整理报告：9.9～19.9 元
- 鲜粮/自制粮个性化配方：9.9～19.9 元

### 订阅层：长期回答"吃什么"

- 全周期饮食管理会员：29 元/月
  - 包含：日常饮食方案、无限动态调整、成分解码不限次、全生命周期管理
- 慢病饮食管理订阅：39 元/月
  - 包含：慢病档案、月度方案、每日跟踪、复查提醒
- 家庭版会员：99 元/月
  - 包含：最多 5 只宠物独立管理

### 远期 B 端层：放大价值

- 为宠物食品品牌提供用户饮食偏好数据
- 为宠物医院提供患者出院后饮食管理工具
- 为保险公司提供健康档案预核保服务

## 7. 已确认与待确认问题

### 已确认

**P0 范围已明确为 6 个核心功能**：
1. 宠物基础信息健康收集
2. 宠物饮食记录
3. 生病周期饮食
4. 现成粮食配置 / 自制鲜粮制作 AI 辅助
5. 粮食推荐和购买
6. 饮食知识库

P1 和 P2 其余内容保持不变。

### 待确认问题

**问题 1：粮食鉴定 / 好物推荐要做到什么程度？**

- A. 轻量做：只提供真伪鉴别指南和中立评测内容
- B. 中度做：接入品牌官方正品查询接口
- C. 重度做：自建鉴定能力，类似得物模式

我们建议选 A，B 可以作为远期选项。C 偏离核心且成本太高。

**问题 2：是否要在 MVP 阶段接品牌合作？**

- A. 不接，先建立中立信任
- B. 接内容合作（品牌赞助科普）
- C. 接产品推荐合作（按效果分成）

我们建议选 A 或 B。C 在 MVP 阶段容易损害中立性。

**问题 3：运动规划是否值得单独作为一个模块？**

- A. 只做 MVP 里的体重记录，运动建议作为饮食方案的一部分
- B. 做一个独立的"宠物 Keep"模块

我们建议选 A，运动作为饮食的联动因素，而不是独立产品。
